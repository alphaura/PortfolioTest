NFT Card Component

NAME OF PROJECT

NFT Card Component

LANGUAGES USED

HTML & CSS

DESCRIPTION

Re-creating an NFT card component with an active status upon hover.

VIEW CHALLENGE ↗ VIEW PROJECT ↗

The Process

This was a simple challenge using HTML & CSS, yet interesting and fun. In this challenge I had to re-create an NFT which showcases the NFT art, number, description, price, time of sale left and the creator. I have used flex-box for most of the project in order to position the images and space the text.

In order to position the `eye` icon over the NFT art upon hover, I decided to go with the CSS position as that allows me to place image over image, by setting the parent box to `relative` and the hover image `absolute` in order to display above it. Understanding when to use flex-box and when to use position was very important in this challenge so that the overall code remains readable and responsive.

The Design

The design was already provided within the FrontEnd Mentor challenge itself, as the aim of these challenges is to replicate the design as much as possible following the information provided upon starting the challenge as images and a READme file are provided as a guide.

The Challenge

The main challenge of this project was placing the image hover affect in the middle. At the time of working on this project, I wasn't fully comfortable with using positioning as I thought that positioning wasn't that ideal, however, after having a breather and took some time away from my laptop, I thought of looking into positioning, which after some research, I came across one of the most common ways to use positioning which is to make the parent container positioned relative and the child component I wanted to make changes to positioned absolute to the parent, and the rest is CSS.
